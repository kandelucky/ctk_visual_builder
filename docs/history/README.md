# Version history

Visual snapshots of CTk Visual Builder across releases.

| Version | Screenshot | Highlights |
|---------|-----------|------------|
| **v0.0.14** | [![v0.0.14](v0.0.14.png)](v0.0.14.png) | **Phase 6.6 — Grid place-based centring + workspace refactor.** CTkFrame's internal canvas (rounded corners + border) broke tk's native `.grid()` centring math — children landed at ¾ of the cell's vertical midpoint. Canvas preview switches to hand-computed `.place()` coordinates that handle every sticky combination (empty = centered, `nsew` = fill, axis-fill, edge-align). Runtime export adds `uniform="col"` / `uniform="row"` on `grid_*configure` so empty cells stay equal-sized (weight=1 alone only splits *extra* space). `Window` loses `layout_type` (absolute positioning only). `WidgetLifecycle` class extracted from `workspace/core.py` (1342 → 998 lines, -26%), handling `widget_added` / `_removed` / `_reparented` / `_z_changed` / `_visibility_changed` / `_locked_changed`. Dead code swept: `_child_grid_kwargs`, `_configure_grid_weights`, `iter_containers` removed. |
| **v0.0.13** | _no screenshot_ | **Phase 6.5 — Grid WYSIWYG + drag-to-cell + runtime parity.** Grid Frames render children into real cells on canvas via tk `.grid()`; user pins grid size via `grid_rows` / `grid_cols` (default 2×2, authoritative — no auto-grow). Drag snaps to the cell under the cursor with a light-blue dashed outline during the gesture; palette drops land at the cursor cell; fresh widgets auto-assign to the next free cell (row-major scan). Runtime export emits `pack_propagate(False)` + `grid_propagate(False)` + `grid_rowconfigure(weight=1)` + `grid_columnconfigure(weight=1)` so runtime matches canvas. `hidden_when` predicate added to the Properties panel — the Dimensions row vanishes on non-grid containers instead of greying out. `project.reorder_child_at(widget_id, final_index)` API for clean drag-to-cell reorders. "Group" palette preset retired (plain Frame + Border toggle does the same). Canvas `[vbox]` / `[hbox]` / `[grid]` badges retired. |
| **v0.0.12** | _no screenshot_ | **Phase 6.4 — Stage 3.1 WYSIWYG + Layout presets + workspace split.** vbox / hbox Frames now render children with real `pack()` on the canvas — the gap between builder preview and exported runtime is closed for the first time. Palette gains a `Layouts` category with four Qt Designer-style presets (`Vertical Layout` / `Horizontal Layout` / `Grid Layout` / `Group`). Child layout collapses from 4 pack_* fields to a single `stretch` enum (`fixed` / `fill` / `grow`), containers gain a `layout_spacing` knob, and `pack_propagate(False)` keeps Frames from shrinking to their children. The 2372-line `workspace.py` also split into a 6-file package (`core` + `chrome` + `controls` + `drag` + `layout_overlay` + `render`), with `core.py` down 48%. Load migration for v0.0.11 projects: legacy `pack_fill` / `pack_expand` translate to the matching `stretch` value. |
| **v0.0.11** | _no screenshot_ | **Phase 6.3 — pack split into vbox / hbox + layout icons.** Matches Qt Designer's 20-year-old UX: `pack` was ergonomically ambiguous so it's been split into `Vertical Layout` (tk `side="top"`) and `Horizontal Layout` (tk `side="left"`). Children drop the `pack_side` row entirely — direction now lives on the parent. Properties panel dropdown renders each option with its Lucide icon (`crosshair` / `rows-3` / `columns-3` / `grid-3x3`); canvas chrome, Object Tree + container badges all updated to the new labels. Legacy v0.0.10 `.ctkproj` files auto-migrate on load (`pack` → `vbox`, or `hbox` if any child had `pack_side=left/right`). Plus: Widget Box palette gained a collapse toggle (icon-only strip when folded). |
| **v0.0.10** | _no screenshot_ | **Phase 6 — Layout managers (stage 1 + 2).** Containers (Window, Frame, ScrollableFrame) gain `layout_type` ∈ `place / pack / grid`; child widgets pick up parent-driven `pack_*` / `grid_*` rows in the Properties panel; code exporter swaps `.place()` for `.pack()` / `.grid()` per parent; canvas shows the choice via chrome-title suffix (`· pack` / `· grid`), `[pack]` / `[grid]` badges in container corners, and dashed grid-cell overlays; Object Tree marks containers with the same suffix. Drag/drop on the canvas is still absolute — true WYSIWYG arrangement is stage 3. |
| **v0.0.9** | [![v0.0.9](v0.0.9.png)](v0.0.9.png) | **Phase 5.5 — Multi-document canvas.** One `.ctkproj` holds a Main Window (`ctk.CTk`) plus any number of Dialogs (`ctk.CTkToplevel`), all visible on the same canvas. Per-document chrome with drag-to-move, active highlight, close ✕, Image widget, palette drop targeting the doc under cursor, cross-document widget drag, smart widget masking for overlapping forms, Object Tree focused on active document with bottom status strip, Window Settings panel (title, size, fg_color live preview, resizable, frameless, builder grid style/colour/spacing), AddDialog preset picker (Alert / Compact / Medium / Settings / Wizard / Same as Main), code exporter emits one `.py` with N classes. |
| **v0.0.8** | [![v0.0.8](v0.0.8.png)](v0.0.8.png) | Phase 3 widgets (13/15 — Entry, CheckBox, ComboBox, OptionMenu, ProgressBar, RadioButton, SegmentedButton, Slider, Switch, Textbox, Tabview, ScrollableFrame). Full Undo/Redo with History panel (F9), Edit menu, coalesced nudges + property scrubs, Reparent / Paste / Duplicate / Z-order / Visibility / Lock toggles, inline rename coalescing. Preview button, Georgian keyboard fallback. |
| **v0.0.7** | [![v0.0.7](v0.0.7.png)](v0.0.7.png) | Properties panel v2 rewrite — ttk.Treeview backbone, flicker-free overlays, modular editor registry. |
| **v0.0.6** | [![v0.0.6](v0.0.6.png)](v0.0.6.png) | CTkLabel / CTkFrame, workspace canvas, startup dialog. |
