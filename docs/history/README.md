# Version history

Visual snapshots of CTk Visual Builder across releases.

| Version | Screenshot | Highlights |
|---------|-----------|------------|
| **v0.0.15.6** | _no screenshot_ | **Workspace refactor round complete — chrome / widget_lifecycle / render / selection_controller / zoom_controller.** Chrome strip gets per-doc `chrome_doc:{id}` umbrella tag — `drive_drag` now slides chrome + doc rect + grid via three `canvas.move` calls instead of the per-motion `_redraw_document` that deleted + recreated ~8 items × N docs per tick. Per-button x-offset constants (`SETTINGS_X_OFFSET` / `TOFRONT_X_OFFSET` / …), `_draw_icon_button` + `_bind_icon_hover` helpers, and a cursor-sticking fix on tofront/toback hover leave. Settings hover bug — was swapping every doc's gear icon at once because the handler used the shared `CHROME_SETTINGS_IMG_TAG` instead of the per-doc tag. `widget_lifecycle.on_widget_added` (120 lines) split into a 55-line orchestrator + 4 helpers (`_resolve_master` / `_instantiate_widget` / `_disable_container_propagate` / `_place_top_level` / `_place_nested`). `render.update_visibility_across_docs` (87 lines, 2 near-duplicate loops) split into a 20-line orchestrator + 4 helpers (`_is_doc_drag_active` / `_compute_doc_bboxes` / `_update_visibility_for_document` / `_should_hide_widget`). `selection_controller` gets the session's biggest perf win — `_edges` (4) + `_handles` (8) + `_outline_pool` frames are allocated once and REUSED across every `draw` / `clear` cycle via `state="hidden"` / `"normal"`, so a stable 20-widget multi-selection costs ~3ms (canvas.coords + itemconfigure) instead of ~45ms (88 tk.Frame destroy+create per draw). `_edge_geoms` extracted as the single source of truth for 4 edge rectangles (previously inlined in 3 sites). `_bbox_for` no longer calls `update_idletasks()` per widget — the public entry points (`draw` / `update`) flush once at the top, saving N-1 redundant flushes per multi-select draw. `zoom_controller.apply_to_widget` gets a `winfo_exists()` guard so the per-call try/except blocks stop swallowing genuine option / geometry errors as "widget gone". Bug fixes: WS-34 (ScrollableFrame `grid_propagate(False)` raised TypeError on load — catch alongside TclError), WS-35 (saving un-hid every hidden widget because `update_visibility_across_docs` wrote `state="normal"` unconditionally — now respects `node.visible`), WS-36 (unhiding a grid child landed it at cell 0,0 — `_set_widget_visibility` now dispatches on parent layout_type and routes grid children through `apply_child_manager`). Doc drag also fixed to call `selection.draw()` (not `update()`) so multi-select outlines follow the document to its new position. |
| **v0.0.15.5** | _no screenshot_ | **Drag perf + layout nesting block + Edit button + locked UX.** Canvas drag motion now hit-tests a press-time cache of grid containers instead of walking every widget — projects with no grid Frames pay zero per-motion cost, projects with a few pay O(grid_count). Layout containers (Vertical / Horizontal / Grid Layout) can no longer be nested inside each other at drop time — palette drop + drag reparent reroute layout-into-layout drops to top-level (WS-33; true nesting deferred to a dedicated mini-phase because canvas rendering of nested grids is fragile). Properties panel gains an "Edit" pencil button in the Name row while Select tool is active: one click narrows a multi-selection to the primary widget and flips the workspace tool to Edit, so users don't have to reach for the toolbar to start editing. Locked widgets are now canvas-inert — clicks on a locked widget (or anywhere in its subtree, since `_effective_locked` cascades to descendants) are silently ignored; the Object Tree is the only surface that can touch them, matching Figma / Photoshop / Illustrator conventions. |
| **v0.0.15.4** | _no screenshot_ | **drag.py refactor + ghost perf + cross-doc group undo.** Drag controller decomposed: `GridDropIndicator` lifted to its own 154-line module; `on_release` (177 → 45-line orchestrator) split into `_teardown_hidden_mode` / `_should_snap_back` / `_apply_snap_back` / `_record_bulk_move`; `_maybe_reparent_dragged` (200 → 70-line orchestrator) split into `_compute_drop_coords` / `_perform_cross_doc_group_move` / `_push_reparent_command`; `on_press` (166 → 65-line orchestrator) split into `_resolve_selection` / `_build_group_starts` / `_tag_drag_group`; `on_motion` (132 → 45-line orchestrator) split into `_motion_feedback` / `_motion_place_group`. Drag ghost for pack/grid children swapped from a per-gesture `tk.Toplevel` (native OS window) to two canvas items (rect + text) tagged `drag_ghost` — each motion is now one `canvas.move` call instead of a Toplevel geometry update. Bug fix: `BulkReparentCommand` added so cross-document group drag records one history entry per moved widget; previously only the primary widget came back on undo and the rest stayed stranded in the destination doc (WS-32). |
| **v0.0.15.3** | _no screenshot_ | **Large-group drag perf + 2 regression fixes.** Group drags of 10+ widgets enter "ghost mode": the widgets + their selection chromes flip to `state="hidden"` on press, a single dashed rectangle tracks the cursor, and `apply_all` runs once on release to drop them at the final position — per-motion tk cost collapses from O(N configure + N font rebuild) to 1 rect move. Document (chrome title bar) drags of 10+ widgets use the same trick: widgets hide, only the doc outline + title bar follow the cursor, then `apply_all` resyncs on release. Bug fixes this pass: (1) `renderer.update_visibility_across_docs` was flipping our intentionally-hidden widgets back to `state="normal"` every motion — now respects a `workspace._doc_drag_hide_active` flag; (2) cross-document widget drag was tripping the off-form snap-back guard because bounds were compared against the *source* doc only — now tests the drop point against every doc's canvas rect. See `docs/testing/workspace.md` WS-30, WS-31. |
| **v0.0.15** | _no screenshot_ | **Area 1 — Workspace core QA pass, 22 bugs fixed.** Cross-document undo/redo now carries `document_id` through `AddWidgetCommand` / `BulkAddCommand` / `DeleteWidgetCommand` / `DeleteMultipleCommand` / `ReparentCommand` so palette drops, paste, duplicate, delete, and drag-reparent all restore into the original doc instead of whichever is active at replay time. `project.add_widget` + `project.reparent` gain a `document_id` param; `project.paste_from_clipboard` gains `base_position` for cursor-anchored paste. Canvas right-click on empty area opens Paste (at cursor) / Select All / Deselect All; right-click on a widget gains Copy / Paste / Duplicate above Rename / Delete / Z-order; multi-select members get `Copy N` / `Duplicate N` / `Delete N` without collapsing the selection. Select All auto-switches Edit→Select tool. Keyboard Delete routes multi-selection through `DeleteMultipleCommand` (locked-guard blocks the whole batch). Tree drag reparent cascade-resets x/y in the new parent and is now fully undoable. Accent colors switch from UUID-hash to doc-index × golden-ratio so two forms can't land on visually-close hues. Left-click on the empty area of a non-active document activates it. Document drag undo/redo re-places widgets via a new `document_position_changed` event. Window Settings opens from the gear icon in Select tool (Hand tool stays strict). See `docs/testing/workspace.md` WS-1..WS-22 for the full finding log. |
| **v0.0.14** | [![v0.0.14](v0.0.14.png)](v0.0.14.png) | **Phase 6.6 — Grid place-based centring + workspace refactor.** CTkFrame's internal canvas (rounded corners + border) broke tk's native `.grid()` centring math — children landed at ¾ of the cell's vertical midpoint. Canvas preview switches to hand-computed `.place()` coordinates that handle every sticky combination (empty = centered, `nsew` = fill, axis-fill, edge-align). Runtime export adds `uniform="col"` / `uniform="row"` on `grid_*configure` so empty cells stay equal-sized (weight=1 alone only splits *extra* space). `Window` loses `layout_type` (absolute positioning only). `WidgetLifecycle` class extracted from `workspace/core.py` (1342 → 998 lines, -26%), handling `widget_added` / `_removed` / `_reparented` / `_z_changed` / `_visibility_changed` / `_locked_changed`. Dead code swept: `_child_grid_kwargs`, `_configure_grid_weights`, `iter_containers` removed. |
| **v0.0.13** | _no screenshot_ | **Phase 6.5 — Grid WYSIWYG + drag-to-cell + runtime parity.** Grid Frames render children into real cells on canvas via tk `.grid()`; user pins grid size via `grid_rows` / `grid_cols` (default 2×2, authoritative — no auto-grow). Drag snaps to the cell under the cursor with a light-blue dashed outline during the gesture; palette drops land at the cursor cell; fresh widgets auto-assign to the next free cell (row-major scan). Runtime export emits `pack_propagate(False)` + `grid_propagate(False)` + `grid_rowconfigure(weight=1)` + `grid_columnconfigure(weight=1)` so runtime matches canvas. `hidden_when` predicate added to the Properties panel — the Dimensions row vanishes on non-grid containers instead of greying out. `project.reorder_child_at(widget_id, final_index)` API for clean drag-to-cell reorders. "Group" palette preset retired (plain Frame + Border toggle does the same). Canvas `[vbox]` / `[hbox]` / `[grid]` badges retired. |
| **v0.0.12** | _no screenshot_ | **Phase 6.4 — Stage 3.1 WYSIWYG + Layout presets + workspace split.** vbox / hbox Frames now render children with real `pack()` on the canvas — the gap between builder preview and exported runtime is closed for the first time. Palette gains a `Layouts` category with four Qt Designer-style presets (`Vertical Layout` / `Horizontal Layout` / `Grid Layout` / `Group`). Child layout collapses from 4 pack_* fields to a single `stretch` enum (`fixed` / `fill` / `grow`), containers gain a `layout_spacing` knob, and `pack_propagate(False)` keeps Frames from shrinking to their children. The 2372-line `workspace.py` also split into a 6-file package (`core` + `chrome` + `controls` + `drag` + `layout_overlay` + `render`), with `core.py` down 48%. Load migration for v0.0.11 projects: legacy `pack_fill` / `pack_expand` translate to the matching `stretch` value. |
| **v0.0.11** | _no screenshot_ | **Phase 6.3 — pack split into vbox / hbox + layout icons.** Matches Qt Designer's 20-year-old UX: `pack` was ergonomically ambiguous so it's been split into `Vertical Layout` (tk `side="top"`) and `Horizontal Layout` (tk `side="left"`). Children drop the `pack_side` row entirely — direction now lives on the parent. Properties panel dropdown renders each option with its Lucide icon (`crosshair` / `rows-3` / `columns-3` / `grid-3x3`); canvas chrome, Object Tree + container badges all updated to the new labels. Legacy v0.0.10 `.ctkproj` files auto-migrate on load (`pack` → `vbox`, or `hbox` if any child had `pack_side=left/right`). Plus: Widget Box palette gained a collapse toggle (icon-only strip when folded). |
| **v0.0.10** | _no screenshot_ | **Phase 6 — Layout managers (stage 1 + 2).** Containers (Window, Frame, ScrollableFrame) gain `layout_type` ∈ `place / pack / grid`; child widgets pick up parent-driven `pack_*` / `grid_*` rows in the Properties panel; code exporter swaps `.place()` for `.pack()` / `.grid()` per parent; canvas shows the choice via chrome-title suffix (`· pack` / `· grid`), `[pack]` / `[grid]` badges in container corners, and dashed grid-cell overlays; Object Tree marks containers with the same suffix. Drag/drop on the canvas is still absolute — true WYSIWYG arrangement is stage 3. |

Per-document chrome with drag-to-move, active highlight, close ✕, Image widget, palette drop targeting the doc under cursor, cross-document widget drag, smart widget masking for overlapping forms, Object Tree focused on active document with bottom status strip, Window Settings panel (title, size, fg_color live preview, resizable, frameless, builder grid style/colour/spacing), AddDialog preset picker (Alert / Compact / Medium / Settings / Wizard / Same as Main), code exporter emits one `.py` with N classes. |
